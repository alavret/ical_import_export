# Перенос календарных событий из Exchange / CalDAV в Яндекс 360

Комплексное решение для миграции календарных событий из Microsoft Exchange или произвольного CalDAV-сервера в Календарь Яндекс 360. Позволяет массово экспортировать события в формат ICS, применить к ним правила модификации и импортировать в календари пользователей Яндекс 360 через CalDAV API.

## Состав решения

Решение состоит из двух независимых компонентов: приложения для экспорта событий из Exchange и Python-скрипта для работы с Календарём Яндекс 360 и внешними CalDAV-серверами.

### 1. Экспорт событий из Exchange

**Каталог:** [`export_ical_by_ews/`](export_ical_by_ews/)

Консольное приложение на C# для массового экспорта календарных событий из Microsoft Exchange в формат ICS через EWS (Exchange Web Services). Поддерживает подключение через Autodiscover или прямой EWS URL, параллельную обработку нескольких ящиков и работу от имени администратора (Full Access или Application Impersonation).

**Ключевые характеристики:**
- Платформа: Windows (.NET 8.0)
- Требования: Exchange 2013+
- Аутентификация: учётная запись с Full Access или ролью ApplicationImpersonation
- Вход: список SMTP-адресов (`input_mailboxes.txt`)
- Выход: ICS-файлы с календарными событиями для каждого ящика

Подробная документация: [export_ical_by_ews/README.md](export_ical_by_ews/README.md)

### 2. Работа с Календарём Яндекс 360 и внешними CalDAV-серверами

**Каталог:** [`caldav_helper/`](caldav_helper/)

Python-скрипт для экспорта и импорта календарных событий в Яндекс 360 через CalDAV API, а также для выгрузки событий из внешних CalDAV-серверов. Поддерживает многопоточную обработку, гибкие правила модификации ICS-файлов и управление транспортными правилами обработки почты.

**Ключевые характеристики:**
- Платформа: Linux, macOS, Windows (Python 3.9+)
- Аутентификация: OAuth-токен Яндекс 360, сервисное приложение
- Вход: список пользователей (`users.csv`), ICS-файлы для импорта
- Выход: ICS-файлы с экспортированными событиями, CSV-отчёты

**Функции (интерактивное меню):**
1. Выгрузка событий из Календаря Яндекс 360
2. Выгрузка событий из внешнего CalDAV-сервера
3. Импорт событий из ICS-файлов в Календарь Яндекс 360
4. Просмотр слоёв календарей пользователей
5. Парсинг ICS-файлов в каталоге (CSV-отчёт)
6. Применение правил модификации к ICS-файлам
8. Настройка правил обработки писем для блокировки уведомлений об удалении событий
9. Настройка сервисного приложения

Подробная документация: [caldav_helper/README.md](caldav_helper/README.md)

## Порядок использования

### 1. Экспорт из Exchange

1. Скопируйте `ExchangeCalendarMigrator.exe`, `config.cfg` и `input_mailboxes.txt` из каталога [`export_ical_by_ews/Publish/`](export_ical_by_ews/Publish/) на Windows-машину с сетевым доступом к Exchange.
2. Укажите в `config.cfg` параметры подключения (EWS URL или Autodiscover, учётные данные).
3. Перечислите ящики для экспорта в `input_mailboxes.txt`.
4. Запустите `ExchangeCalendarMigrator.exe`.
5. ICS-файлы с событиями будут сохранены в каталог `output_dir`.

### 2. Экспорт из внешнего CalDAV-сервера (альтернативный источник)

1. Установите зависимости: `pip install -r requirements.txt` (в каталоге `caldav_helper/`).
2. Настройте файл `.env`: укажите `EXTERNAL_CALDAV_URL` и путь к `external_caldav_users.csv`.
3. Заполните `external_caldav_users.csv` (формат: `alias;login;password`).
4. Запустите `python y360_calendar.py` и выберите пункт `2` (выгрузка из внешнего CalDAV).
5. Укажите пользователей и диапазон дат.
6. ICS-файлы будут сохранены в каталог `output/`.

### 3. Применение правил модификации (опционально)

Перед импортом можно преобразовать события с помощью правил из файла `ical_modify_rules.txt`:

1. Опишите правила модификации в файле `ical_modify_rules.txt` (формат: `тег;оператор;значение1;значение2`).
2. Запустите `python y360_calendar.py` и выберите пункт `6`.
3. Укажите каталог с исходными ICS-файлами и каталог для результатов.

Поддерживаемые правила:

| Оператор | Описание | Пример |
|----------|----------|--------|
| `replace` | Замена значения тега | `summary;replace;Совещание;Встреча` |
| `delete` | Удаление участника по email | `*;delete;user@domain.ru` |
| `add` | Добавление участника | `*;add;user@domain.ru;Имя` |

### 4. Импорт в Яндекс 360

1. Настройте файл `.env`: укажите `OAUTH_TOKEN`, `ORG_ID`, `SERVICE_APP_ID`, `SERVICE_APP_SECRET`.
2. Настройте сервисное приложение (пункт меню `9`).
3. Скопируйте ICS-файлы в каталог `input/`.
4. Заполните `users.csv` списком пользователей для импорта.
5. Запустите `python y360_calendar.py` и выберите пункт `3` (импорт).
6. Выберите пользователей, укажите диапазон дат и политику обработки дубликатов.

### 5. Блокировка уведомлений об удалении (рекомендуется)

При удалении событий из Календаря Яндекс 360 через CalDAV участникам рассылаются уведомления об отмене. Чтобы предотвратить это:

1. Запустите `python y360_calendar.py` и выберите пункт `8`.
2. Выберите пункт `3` — добавить транспортное правило для блокировки уведомлений.

## Конфигурация caldav_helper

### Файл .env

| Параметр | Описание | Пример |
|----------|----------|--------|
| `OAUTH_TOKEN` | OAuth-токен для доступа к API Яндекс 360 | `y0__xxxxx` |
| `ORG_ID` | ID организации в Яндекс 360 | `1234567` |
| `USERS_FILE` | Путь к файлу со списком пользователей | `users.csv` |
| `DRY_RUN` | Режим пробного запуска (`true`/`false`) | `False` |
| `SERVICE_APP_ID` | ID сервисного приложения Яндекс 360 | `f1e4386f276643` |
| `SERVICE_APP_SECRET` | Секрет сервисного приложения | `333bf50d7a8` |
| `VERBOSE_LOGGING` | Подробное логирование (`true`/`false`) | `false` |
| `RULE_APPLY_REPORT` | Путь к отчёту о применении правил | `rule_apply.csv` |
| `ROUTING_RULES_FILE` | Путь к файлу с транспортными правилами | `routing_rules.json` |
| `CREATE_CANCEL_RULES_FOR_EVENTS_DELETIONS` | Создавать правило блокировки уведомлений при удалении событий | `true` |
| `EXTERNAL_CALDAV_URL` | URL внешнего CalDAV-сервера | `https://caldav.server.ru/caldav` |
| `EXTERNAL_CALDAV_USERS_FILE` | Путь к файлу с пользователями внешнего CalDAV | `external_caldav_users.csv` |

### Файл users.csv

Список email-адресов пользователей Яндекс 360:

```
email
user01@example.com
user02@example.com
```

### Файл external_caldav_users.csv

Учётные данные для подключения к внешнему CalDAV-серверу (разделитель `;`):

```
alias;login;password
user1;user1@domain.ru;P@ssw0rd!123
user2;user2@domain.ru;P@ssw0rd!123
```
